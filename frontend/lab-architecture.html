Mục đích trang này

Giải thích thành phần hệ thống, đường đi dữ liệu từ trình duyệt tới PostgreSQL và file mock lịch, và mối liên hệ với LangGraph (chi tiết từng nút xem Sơ đồ LangGraph).

2. Một tin nhắn chat — data flow (SSE)

Luồng thực tế trong POST /chat/sessions/{id}/messages (chat.py): lưu tin nhắn người dùng → chạy graph → stream sự kiện (status / token / done).

sequenceDiagram autonumber participant U as Trình duyệt participant API as FastAPI chat participant DB as PostgreSQL participant G as LangGraph participant Tool as Tools Python participant F as File mock JSON U->>API: POST message (Form + Bearer trong form) API->>DB: INSERT tin nhắn user API->>G: astream_events(input_state + thread_id=session_id) G->>G: classify_intent … (các nút theo intent) G->>Tool: save_consult_intake / get_mock_schedule / book_appointment Tool->>DB: INSERT/UPDATE intake, reservation Tool->>F: Đọc lich_trong_tuan_trong_vi.json G-->>API: on_chain_* / on_chat_model_stream API-->>U: SSE: status, token, done
  • Checkpoint Redis (langgraph-checkpoint-redis): state graph theo thread_id = session_id — giữ nguyên sau khi restart server.
  • Tin nhắn lịch sử chat lưu bảng message; state graph là tầng riêng cho agent.

5. Bảng chính PostgreSQL (khái niệm)

  • patient_users — tài khoản BN, JWT sub
  • booking_chat_sessions + booking_chat_messages — hội thoại UI
  • booking_consult_intakes — tóm tắt triệu chứng sau specialist, category_code
  • reservations — lịch đã đặt, liên kết intake

6. Phòng lab (dev)

admin.html — gọi trực tiếp từng node agent hoặc tool, không cần JWT cho hầu hết tool/lab. REST nhanh trên cùng trang để đối chiếu với file mock (cột phải «Dữ liệu mock»).

API bổ sung: GET /api/v1/admin/lab/triage-rubric — dump JSON rubric; GET …/mock-schedule-summary — tóm tắt slot theo tuần.

Ghi chú vận hành

  • Service Redis Stack trong Docker Compose là bắt buộc — backend dùng langgraph-checkpoint-redis để persist graph state. Nếu Redis không sẵn sàng, fallback MemorySaver (mất state khi restart).
  • Upload ảnh / Pillow: cấu hình UPLOAD_DIR còn trong settings; API chat hiện chỉ nhận văn bản (Form field message).
